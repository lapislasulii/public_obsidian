# Конспект лекции 19: Амортизационный анализ


---

## 1. Обзор лекции

Лекция посвящена амортизационному анализу — методу оценки временной сложности последовательности операций в структурах данных. В начале занятия кратко повторяется алгоритм Дейкстры с эвристикой (A\*) и алгоритм Беллмана-Форда для графов с отрицательными весами рёбер, что создаёт контекст для понимания, зачем вообще нужна точная оценка сложности алгоритмов. Основной акцент делается на том, что стандартный анализ «наихудшего случая» может давать пессимистичную и нереалистичную оценку при разнородных операциях. Разбираются три метода амортизационного анализа: групповой анализ, метод бухгалтерского учёта и метод потенциалов. На конкретных примерах (инкремент бинарной маски и динамическая вставка в массив) показывается, как получить амортизационную оценку $O^*(1)$ вместо кажущейся $O(n)$.

---

## 2. Предварительные сведения

- **Асимптотические обозначения:** $O$, $\Theta$, $\Omega$ — верхняя, точная и нижняя оценки сложности.
- **Алгоритм Дейкстры:** жадный обход графа с неотрицательными весами, сложность $O((V + E) \log V)$ с бинарной кучей.
- **Алгоритм Беллмана-Форда:** динамическое программирование, обрабатывает отрицательные рёбра, сложность $O(V \cdot E)$.
- **Алгоритм Флойда-Уоршалла:** все пары кратчайших путей, сложность $O(V^3)$.
- **Динамический массив (вектор):** структура данных, автоматически расширяющая ёмкость (`capacity`) при добавлении элементов.
- **Связный список (linked list):** структура данных, вставка за $O(1)$, но нет произвольного доступа.
- **Хеш-таблица:** структура с амортизированной вставкой $O(1)$ при правильном выборе коэффициента загрузки.
- **Бинарная маска:** способ представления подмножества или состояния в виде битового массива из $N$ бит.

## 3. Основное содержание

### 3.1 Краткое повторение: алгоритм Дейкстры и A*

#### Дейкстра

**Определение.** Алгоритм Дейкстры находит кратчайшие пути от одной исходной вершины до всех остальных в графе с **неотрицательными** весами рёбер.

**Механизм работы:**
1. Инициализировать расстояние до исходной вершины $= 0$, до остальных $= \infty$.
2. Добавить исходную вершину в приоритетную очередь.
3. Пока очередь не пуста: извлечь вершину с минимальным расстоянием (pop), обновить расстояния до её соседей по формуле $d[v] = d[u] + w(u, v)$, добавить обновлённые вершины в очередь.

**Сложность:**

| Реализация очереди | Сложность |
|--------------------|-----------|
| Массив / список    | $O(V^2)$ |
| Бинарная куча      | $O((V + E) \log V)$ |
| Куча Фибоначчи     | $O(E + V \log V)$ |

**Почему жадность работает:** алгоритм всегда обрабатывает вершину с наименьшим текущим расстоянием — при неотрицательных рёбрах это расстояние уже является финальным.

**Типичная ошибка:** применение Дейкстры к графу с **отрицательными** рёбрами. Из-за жадного выбора минимума алгоритм может пропустить более короткий путь, который проходит через «дешёвое» отрицательное ребро позже.

#### Алгоритм A* (A со звёздочкой)

**Определение.** Модификация Дейкстры с эвристической функцией $h(v)$, которая оценивает расстояние от текущей вершины до цели. Приоритет в очереди задаётся как $f(v) = d[v] + h(v)$.

**Требование к эвристике (допустимость):** $h(v) \leq$ реальное расстояние до цели. Если эвристика нарушает это условие, алгоритм может дать неверный ответ.

**Пример:** навигация по карте — эвристика евклидово расстояние (по широте-долготе). При учёте рельефа (высоты) можно строить эвристику на основе угла подъёма между точками.

**Дополнено:** A* применим только тогда, когда в узлах графа есть дополнительные атрибуты (координаты, высота и т.д.), позволяющие посчитать осмысленную эвристику. На произвольном графе без метрики A* вырождается в Дейкстру.[/дополнено]

### 3.2 Алгоритм Беллмана-Форда

**Определение.** Алгоритм поиска кратчайших путей от одной исходной вершины, корректно обрабатывающий **отрицательные веса рёбер** и обнаруживающий **отрицательные циклы**.

**Механизм работы:**
1. Создать массив `dist` размером $V$: `dist[source] = 0`, остальные $= \infty$.
2. Выполнить $V - 1$ итераций релаксации по **всем** рёбрам:
   $$\text{dist}[v] = \min(\text{dist}[v],\ \text{dist}[u] + w(u, v))$$
3. Выполнить ещё одну ($V$-ю) итерацию: если какое-либо расстояние обновляется — в графе есть **отрицательный цикл**.

**Ключевое отличие от Дейкстры:** нет жадного выбора — каждую итерацию обходятся все рёбра. Именно поэтому алгоритм не ошибается при отрицательных рёбрах.

**Сложность:** $O(V \cdot E)$

**Почему значения обновляются «до» итерации, а не сразу:** на каждой итерации $i$ алгоритм находит кратчайшие пути, использующие не более $i$ рёбер. Если обновлять `dist` немедленно в той же итерации, можно «перепрыгнуть» через несколько рёбер за одну итерацию и получить некорректный результат для задачи обнаружения отрицательных циклов.

**Обнаружение отрицательного цикла:** если на $V$-й итерации значение в массиве `dist` продолжает уменьшаться — значит, в графе есть цикл с суммарным отрицательным весом. Для маленького цикла это видно сразу (значение уходит в $-\infty$ быстро), для большого — после большего числа итераций.

```python
def bellman_ford(graph, source):
    dist = {v: float('inf') for v in graph.vertices}
    dist[source] = 0

    for _ in range(len(graph.vertices) - 1):
        for u, v, w in graph.edges:
            if dist[u] + w < dist[v]:
                dist[v] = dist[u] + w

    # Проверка отрицательного цикла
    for u, v, w in graph.edges:
        if dist[u] + w < dist[v]:
            return None  # Отрицательный цикл обнаружен

    return dist
```

**Типичная ошибка:** забывать про $V$-ю дополнительную итерацию для обнаружения отрицательных циклов.

### 3.3 Алгоритм Флойда-Уоршалла (краткое упоминание)

**Определение.** Алгоритм нахождения кратчайших путей между **всеми парами** вершин графа, основанный на динамическом программировании.

**Сложность:** $O(V^3)$

**Реализация:** использует матрицу расстояний $V \times V$ и тройной цикл по промежуточным вершинам $k$, начальным $i$ и конечным $j$:
$$\text{dist}[i][j] = \min(\text{dist}[i][j],\ \text{dist}[i][k] + \text{dist}[k][j])$$

На практике применяется редко из-за кубической сложности; упоминается как учебный пример DP на графах.

### 3.4 Что такое амортизационный анализ

**Определение.** Амортизационный анализ — метод анализа сложности **последовательности** разнородных операций над структурой данных, дающий **среднюю стоимость одной операции** по всей последовательности, а не оценку наихудшего случая для каждой операции в отдельности.

Обозначение: $O^*(f(n))$ — амортизационная сложность.

**Зачем нужен:**
- Некоторые операции над структурой данных иногда «дорогие» ($O(n)$), но чаще «дешёвые» ($O(1)$). Стандартная оценка наихудшего случая даёт $O(n)$ на операцию, что пессимистично.
- Амортизационный анализ позволяет показать, что суммарная стоимость $n$ операций равна $O(n)$, то есть **в среднем** — $O(1)$ на операцию.

**Примеры применения:**
- Динамическая вставка в массив (вектор) с расширением ёмкости.
- Перестройка динамического дерева отрезков.
- Вставка в хеш-таблицу (рехеширование).
- Кеширование данных.
- Инкремент двоичного счётчика.
- В машинном обучении: подгрузка данных для нейросети в распределённых системах; пересчёт центров масс в кластеризации.

**Интуиция:** как в банке — «дешёвые» операции делают «взносы» на счёт, из которого «дорогие» операции покрывают своё превышение стоимости. Если счёт никогда не уходит в минус, значит, суммарная стоимость ограничена суммой «взносов».

**Важное замечание:** амортизационная оценка — это **гарантированная средняя** оценка наихудшего случая для последовательности, а **не** вероятностная (средняя по входным данным). Это принципиальное отличие от среднего случая в вероятностном анализе.

### 3.5 Три метода амортизационного анализа

#### Обзор методов

| Метод | Идея | Формула | Сложность применения |
|-------|------|---------|----------------------|
| Групповой анализ | Посчитать суммарную стоимость $n$ операций, поделить на $n$ | $a^* = T(n) / n$ | Простой, но не всегда применим |
| Бухгалтерский учёт | Назначить «цены» операциям, следить, чтобы «банк» не уходил в минус | $\sum a_i \geq \sum t_i$ | Средний, требует выбора цен |
| Метод потенциалов | Ввести функцию потенциала структуры, выразить через неё амортизированную стоимость | $a_i = t_i + \Phi(S_i) - \Phi(S_{i-1})$ | Сложнее, но наиболее мощный |

---

### 3.6 Метод 1: Групповой (агрегатный) анализ

**Определение.** Подсчитывается **суммарная** реальная стоимость $T(n)$ всех $n$ операций в последовательности. Амортизированная стоимость одной операции:

$$a^* = \frac{T(n)}{n}$$

**Пример 1: Инкремент бинарного счётчика (маски)**

Задача: есть $N$-битный двоичный счётчик. Операция `increment` прибавляет 1. Сколько битовых переходов происходит за $2^N$ последовательных инкрементов?

Алгоритм `increment` идёт справа налево, инвертируя единицы в нули, пока не встретит 0 (который инвертирует в 1 и останавливается).

Анализ по позициям битов:
- Бит 0 (самый правый) переключается при **каждом** инкременте: $2^N$ раз.
- Бит 1 переключается каждые 2 инкремента: $2^{N-1}$ раз.
- Бит 2 переключается каждые 4 инкремента: $2^{N-2}$ раз.
- ...
- Бит $k$ переключается $2^{N-k}$ раз.

Суммарное число переключений за $2^N$ операций:

$$T(2^N) = \sum_{k=0}^{N-1} 2^{N-k} = 2^N + 2^{N-1} + \ldots + 2 = 2^{N+1} - 2 < 2 \cdot 2^N$$

Амортизированная стоимость одной операции:

$$a^* = \frac{T(2^N)}{2^N} < 2 = O(1)$$

**Вывод:** несмотря на то что в худшем случае `increment` стоит $O(N)$ (все биты — единицы), амортизированная стоимость равна $O^*(1)$.

**Интуиция:** «дорогие» вызовы (много единиц) случаются редко — после такого вызова почти все биты обнуляются, и следующие инкременты снова «дёшевы».

**Типичная ошибка:** принять за амортизированную оценку $O(N)$ (наихудший случай одного вызова), не учитывая редкость дорогих случаев.

---

**Пример 2: Динамическая вставка в массив (групповой анализ)**

Задача: добавляем $n$ элементов в динамический массив. Два варианта расширения:
- **Вариант A:** при заполнении увеличивать `capacity` на 1 (т.е. `capacity = capacity + 1`).
- **Вариант B:** при заполнении удваивать `capacity` (т.е. `capacity = capacity * 2`).

**Вариант A (линейное расширение):**

При каждом расширении нужно скопировать все текущие $n$ элементов. Расширения происходят на каждом шаге:

$$T(n) = 1 + 2 + 3 + \ldots + n = \frac{n(n+1)}{2} = O(n^2)$$

Амортизированная стоимость: $a^* = O(n^2)/n = O(n)$ на операцию.

**Вариант B (удвоение ёмкости):**

Расширения происходят на шагах $1, 2, 4, 8, \ldots, 2^k$, где $2^k \leq n$. Стоимость каждого расширения равна числу скопированных элементов:

$$T(n) = 1 + 2 + 4 + \ldots + n \approx 2n = O(n)$$

Это геометрическая прогрессия: суммарная стоимость $O(n)$.

Амортизированная стоимость: $a^* = O(n)/n = O(1)$ на операцию.

**Вывод:**

| Стратегия расширения | Суммарная стоимость | Амортизированная стоимость |
|----------------------|--------------------|-----------------------------|
| `capacity + 1` (на 1) | $O(n^2)$ | $O(n)$ на операцию |
| `capacity * 2` (удвоение) | $O(n)$ | $O^*(1)$ на операцию |

**Типичная ошибка:** увеличивать ёмкость на 1 каждый раз — интуитивно «экономно», но катастрофично по сложности.

### 3.7 Метод 2: Бухгалтерский учёт (метод монеток)

**Определение.** Каждой операции назначается **амортизированная стоимость** $a_i$, которая может отличаться от реальной стоимости $t_i$. Разница $a_i - t_i$ — это «кредит», который сохраняется в «банке» структуры данных для оплаты будущих дорогих операций.

**Инвариант корректности:** банк никогда не должен уходить в минус:
$$\sum_{i=1}^{k} a_i \geq \sum_{i=1}^{k} t_i \quad \text{для всех } k$$

Если инвариант соблюдается, то суммарная реальная стоимость ограничена суммой амортизированных стоимостей:
$$\sum t_i \leq \sum a_i$$

**Пример: Инкремент бинарного счётчика (метод монеток)**

Операции над битами:
- Изменить $0 \to 1$: реальная стоимость $= 1$.
- Изменить $1 \to 0$: реальная стоимость $= 1$.

**Назначение амортизированных стоимостей:**
- Изменить $0 \to 1$: амортизированная стоимость $= 2$ (платим 1 за саму операцию, ещё 1 кладём в «банк» при этом бите).
- Изменить $1 \to 0$: амортизированная стоимость $= 0$ (платим из «банка» монетку, которую положили при $0 \to 1$).

**Логика:** когда мы устанавливаем бит в 1, мы «предоплачиваем» его будущий сброс. Поэтому сброс единицы в ноль бесплатен — монетка уже есть в банке.

**Пример трассировки ($N = 4$):**

| Состояние | Операция `increment` | Монеток в банке |
|-----------|----------------------|----------------|
| `0000` | — | 0 |
| `0001` | $0 \to 1$ (бит 0) | 1 |
| `0010` | $1 \to 0$ (бит 0), $0 \to 1$ (бит 1) | 1 |
| `0011` | $0 \to 1$ (бит 0) | 2 |
| `0100` | $1 \to 0$ (бит 0, 1), $0 \to 1$ (бит 2) | 1 |
| `0101` | $0 \to 1$ (бит 0) | 2 |
| `0110` | $1 \to 0$ (бит 0), $0 \to 1$ (бит 1) | 2 |
| `0111` | $0 \to 1$ (бит 0) | 3 |
| `1000` | $1 \to 0$ (биты 0, 1, 2), $0 \to 1$ (бит 3) | 1 |

Банк никогда не уходит в минус. Каждая операция `increment` стоит амортизированно $\leq 2 = O(1)$.

**Интуиция:** монетки, «отложенные» при установке бита в 1, покрывают стоимость его сброса. Сколько бы единиц ни было в числе перед инкрементом, у каждой из них уже есть своя монетка.

**Типичная ошибка:** назначить амортизированную стоимость $1$ для $0 \to 1$ (не откладывать «предоплату»). Тогда банк уйдёт в минус при длинной цепочке сбросов.

---

**Пример: Динамическая вставка в массив (метод монеток)**

Операция `append`:
- **Дешёвый случай:** просто записать элемент на следующую позицию (без расширения). Реальная стоимость $= 1$.
- **Дорогой случай:** выделить новый массив вдвое большего размера, скопировать $n$ элементов, добавить новый. Реальная стоимость $= n + 1 \approx n$.

**Назначение амортизированных стоимостей при удвоении:**
- Каждой операции `append` назначаем амортизированную стоимость $= 3$:
  - 1 монетка — за запись нового элемента.
  - 1 монетка — «предоплата» за копирование этого нового элемента при следующем расширении.
  - 1 монетка — «предоплата» за копирование одного из «старых» элементов, которые будут перенесены.

**Инвариант:** перед расширением в «банке» накоплено $\geq n/2$ монеток (по одной от каждого из $n/2$ элементов, добавленных после предыдущего расширения). Этого достаточно, чтобы оплатить копирование всех $n$ элементов.

**Вывод:** амортизированная стоимость `append` равна $O^*(1)$, несмотря на то что расширение стоит $O(n)$.

### 3.8 Метод 3: Метод потенциалов

**Определение.** Вводится функция потенциала $\Phi(S)$, отображающая текущее состояние структуры данных $S$ в неотрицательное число. Амортизированная стоимость операции определяется как:

$$a_i = t_i + \Phi(S_i) - \Phi(S_{i-1})$$

**Обязательное условие:** $\Phi(S_i) \geq \Phi(S_0)$ для всех $i$ (потенциал не падает ниже начального).

Из этого условия следует:

$$\sum_{i=1}^{n} a_i = \sum_{i=1}^{n} t_i + \Phi(S_n) - \Phi(S_0) \geq \sum_{i=1}^{n} t_i$$

Значит, сумма амортизированных стоимостей — верхняя граница суммарной реальной стоимости.

**Пример: Инкремент бинарного счётчика (метод потенциалов)**

**Выбор функции потенциала:**
$$\Phi(S) = \text{число единичных битов в текущем состоянии счётчика}$$

Начальное состояние: $\Phi(S_0) = 0$. Условие $\Phi(S_i) \geq 0$ выполнено всегда.

**Расчёт амортизированной стоимости:**

Пусть операция `increment` устанавливает $k$ единичных битов в 0 и один $0$-бит в 1 (т.е. «сбрасывает» $k$ единиц и «поднимает» одну).

- Реальная стоимость: $t_i = k + 1$ (столько битовых переходов).
- Изменение потенциала: $\Phi(S_i) - \Phi(S_{i-1}) = 1 - k$ (один бит стал 1, $k$ битов стали 0).
- Амортизированная стоимость: $a_i = (k + 1) + (1 - k) = 2 = O(1)$.

**Трассировка ($N = 4$, показана функция $\Phi = \text{sum}(1)$ — число единичных битов):**

| Состояние | $\Phi(S)$ | $\Delta\Phi$ | $t_i$ | $a_i = t_i + \Delta\Phi$ |
|-----------|-----------|--------------|-------|--------------------------|
| `0000` | 0 | — | — | — |
| `0001` | 1 | +1 | 1 | 2 |
| `0010` | 1 | 0 | 2 | 2 |
| `0011` | 2 | +1 | 1 | 2 |
| `0100` | 1 | −1 | 4 | 3 → 2* |
| `0101` | 2 | +1 | 1 | 2 |
| `0110` | 2 | 0 | 2 | 2 |
| `0111` | 3 | +1 | 1 | 2 |
| `1000` | 1 | −2 | 4 | 2 |

*В точных вычислениях всегда получается $\leq 2$.

**Вывод:** для любого инкремента $a_i = 2 = O(1)$.

**Интуиция:** потенциал «накапливает» стоимость будущих сбросов. Когда дорогой инкремент «тратит» много сбросов, потенциал резко падает, и амортизированная стоимость остаётся маленькой.

---

**Дополнено:** Выбор функции потенциала — ключевой навык. Хорошая функция потенциала должна расти при «дешёвых» операциях и падать при «дорогих», компенсируя их стоимость. Плохой выбор $\Phi$ либо не даёт нужной оценки, либо нарушает условие неотрицательности.[/дополнено]

### 3.9 Сравнение стратегий расширения динамического массива

Подробный разбор двух стратегий на конкретных значениях:

**Обозначения:** $n$ — текущий размер (size), $m$ — ёмкость (capacity), $a_k$ — стоимость $k$-й операции `append`.

**Стратегия 1: увеличение на 1 ($m \leftarrow m + 1$)**

| Шаг $k$ | $n$ (до) | $m$ (до) | Операция | Стоимость $a_k$ |
|---------|----------|----------|----------|-----------------|
| 1 | 0 | 1 | просто вставить | 1 |
| 2 | 1 | 1 | расширить до 2, скопировать 1, вставить | $1 + 1 = 2$ |
| 3 | 2 | 2 | расширить до 3, скопировать 2, вставить | $2 + 1 = 3$ |
| 4 | 3 | 3 | расширить до 4, скопировать 3, вставить | $3 + 1 = 4$ |
| ... | ... | ... | ... | ... |
| $k$ | $k-1$ | $k-1$ | расширить, скопировать $k-1$, вставить | $k$ |

Суммарная стоимость: $T(n) = 1 + 2 + 3 + \ldots + n = \frac{n(n+1)}{2} = O(n^2)$

Амортизированная стоимость: $a^* = O(n)$ — **плохо**.

**Стратегия 2: удвоение ($m \leftarrow 2m$)**

| Шаг $k$ | $n$ (до) | $m$ (до) | Операция | Стоимость $a_k$ |
|---------|----------|----------|----------|-----------------|
| 1 | 0 | 1 | просто вставить | 1 |
| 2 | 1 | 1 | расширить до 2, скопировать 1, вставить | 2 |
| 3 | 2 | 2 | расширить до 4, скопировать 2, вставить | 3 |
| 4 | 3 | 4 | просто вставить | 1 |
| 5 | 4 | 4 | расширить до 8, скопировать 4, вставить | 5 |
| 6 | 5 | 8 | просто вставить | 1 |
| 7 | 6 | 8 | просто вставить | 1 |
| 8 | 7 | 8 | просто вставить | 1 |
| 9 | 8 | 8 | расширить до 16, скопировать 8, вставить | 9 |

Суммарная стоимость: $T(n) \approx n + n/2 + n/4 + \ldots = 2n = O(n)$ (геометрическая прогрессия)

Амортизированная стоимость: $a^* = O(1)$ — **хорошо**.

## 6. Справочный лист

### Ключевые определения

- **Амортизированная сложность $O^*(f(n))$:** гарантированная средняя стоимость одной операции при выполнении последовательности из $n$ операций в наихудшем случае.
- **Потенциал $\Phi(S)$:** неотрицательная функция состояния структуры данных; $\Phi(S_0) = 0$, $\Phi(S_i) \geq 0$.
- **Кредит:** разница $a_i - t_i > 0$, накапливаемая при «дешёвых» операциях для покрытия «дорогих».

### Формулы

| Метод | Формула |
|-------|---------|
| Групповой | $a^* = T(n)/n$ |
| Бухгалтерский | $a_i \geq t_i$; инвариант: $\sum a_i \geq \sum t_i$ |
| Потенциалов | $a_i = t_i + \Phi(S_i) - \Phi(S_{i-1})$; $\Phi(S_i) \geq \Phi(S_0) = 0$ |

### Ключевые результаты

| Задача | Наихудший случай | Амортизированный |
|--------|-----------------|-----------------|
| Инкремент $N$-битного счётчика | $O(N)$ | $O^*(1)$ |
| Вставка в динамический массив (×2) | $O(n)$ | $O^*(1)$ |
| Вставка в динамический массив (+1) | $O(n)$ | $O(n)$ — не улучшается |
| Вставка в хеш-таблицу (при рехешировании) | $O(n)$ | $O^*(1)$ |

### Алгоритмы на графах — сводка сложностей

| Алгоритм | Применение | Сложность |
|----------|-----------|-----------|
| Дейкстра (двоичная куча) | Кратчайший путь (неотриц. веса) | $O((V+E)\log V)$ |
| A* | Кратчайший путь с эвристикой | Зависит от эвристики |
| Беллман-Форд | Кратчайший путь (отриц. веса, обнаружение циклов) | $O(V \cdot E)$ |
| Флойд-Уоршалл | Все пары кратчайших путей | $O(V^3)$ |

## 7. Практические советы

1. **Выбирайте стратегию удвоения для динамических массивов.** Увеличение ёмкости на 1 при каждом переполнении приводит к амортизированной сложности $O(n)$ на вставку, что делает добавление $n$ элементов операцией $O(n^2)$. Удвоение даёт $O^*(1)$.

2. **Используйте групповой анализ как первый шаг.** Это самый простой метод: посчитайте точную суммарную стоимость $T(n)$ и поделите на $n$. Если это трудно посчитать аналитически — переходите к бухгалтерскому учёту или методу потенциалов.

3. **При выборе функции потенциала ищите «накопитель» дорогих операций.** Потенциал должен расти при дешёвых операциях и резко падать при дорогих, компенсируя их реальную стоимость. Для бинарного счётчика — число единичных битов. Для динамического массива — число элементов во «второй половине» (добавленных после последнего расширения).

4. **Не путайте амортизированный анализ с анализом среднего случая.** Амортизированная оценка — это **гарантия для наихудшей** последовательности операций, а не ожидаемое время при случайных входных данных. Она всегда корректна, тогда как среднее время зависит от распределения входных данных.

5. **Помните об амортизации при выборе структур данных.** Если ваш алгоритм выполняет $n$ вставок в динамический массив, реальная суммарная сложность — $O(n)$, а не $O(n^2)$. Именно поэтому `std::vector` (C++) и `list` (Python) так эффективны на практике — их амортизированная сложность вставки $O^*(1)$, а не $O(n)$.

## Приложение: Иллюстративные реализации

### Инкремент бинарного счётчика

```python
def increment(mask: list[int]) -> list[int]:
    """
    Инкремент N-битного двоичного счётчика (список битов, старший — слева).
    Идём справа налево: сбрасываем единицы, пока не встретим ноль.
    Худший случай: O(N), амортизированный: O*(1).
    """
    n = len(mask)
    i = n - 1  # Начинаем с младшего бита
    while i >= 0 and mask[i] == 1:
        mask[i] = 0  # 1 -> 0 (стоит 1 операцию, оплачивается монеткой из банка)
        i -= 1
    if i >= 0:
        mask[i] = 1  # 0 -> 1 (стоит 1 операцию + кладём 1 монетку в банк)
    return mask
```

### Динамический массив с удвоением

```python
class DynamicArray:
    def __init__(self):
        self._capacity = 1
        self._size = 0
        self._data = [None] * self._capacity

    def append(self, value):
        """
        Амортизированная сложность: O*(1).
        При переполнении: расширяем в 2 раза и копируем — O(n), но редко.
        """
        if self._size == self._capacity:
            # Дорогая операция: выделяем новый массив и копируем
            self._capacity *= 2
            new_data = [None] * self._capacity
            for i in range(self._size):
                new_data[i] = self._data[i]
            self._data = new_data

        # Дешёвая операция: просто записываем элемент
        self._data[self._size] = value
        self._size += 1

    def __len__(self):
        return self._size
```

### Алгоритм Беллман-Форда с обнаружением отрицательного цикла

```python
def bellman_ford(n_vertices, edges, source):
    """
    n_vertices: количество вершин
    edges: список рёбер (u, v, weight)
    source: исходная вершина
    
    Сложность: O(V * E)
    Возвращает: массив расстояний или None при наличии отрицательного цикла.
    """
    dist = [float('inf')] * n_vertices
    dist[source] = 0

    # V - 1 итераций релаксации
    for iteration in range(n_vertices - 1):
        updated = False
        for u, v, w in edges:
            if dist[u] != float('inf') and dist[u] + w < dist[v]:
                dist[v] = dist[u] + w
                updated = True
        if not updated:
            break  # Ранний выход, если изменений нет

    # Проверка отрицательного цикла (V-я итерация)
    for u, v, w in edges:
        if dist[u] != float('inf') and dist[u] + w < dist[v]:
            return None  # Обнаружен отрицательный цикл

    return dist
```
