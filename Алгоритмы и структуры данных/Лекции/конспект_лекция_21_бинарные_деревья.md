# Конспект лекции 21: Бинарные деревья поиска и красно-чёрные деревья

---

## 1. Обзор лекции

Лекция охватывает два фундаментальных раздела: **бинарные деревья поиска (BST)** и **красно-чёрные деревья (RBT)** как способ их балансировки. Разбираются все ключевые операции над BST — поиск, обходы, вставка, удаление — и доказывается, что математическое ожидание высоты случайно построенного BST равно $O(\lg n)$. Далее вводятся красно-чёрные деревья с пятью инвариантами цвета и доказывается лемма об ограничении высоты сверху величиной $2\lg(n+1)$. Отдельно лектор затрагивает структуру данных DSU (система непересекающихся множеств) как важный смежный инструмент, активно используемый в графовых алгоритмах (например, алгоритм Краскала). В конце обсуждаются эвристики сжатия пути и объединения по рангу, дающие почти константную амортизационную сложность.

---

## 2. Предварительные сведения

- **Рекурсия и стек вызовов** — обходы дерева реализованы рекурсивно.
- **Указатели и динамическая память** — каждый узел хранит указатели на родителя, левого и правого потомка.
- **Понятие высоты дерева** ($h$) — длина наидлиннейшего пути от корня до листа.
- **Асимптотический анализ** — нотации $O$, $\Theta$, $\Omega$; амортизационная сложность.
- **Хэш-таблицы и множества** — используются в DSU для быстрого поиска представителя.
- **Математическое ожидание и индикаторные случайные величины** — нужны для доказательства теоремы 12.4.
- **Понятие сбалансированного дерева** — дерево сбалансировано, если $h = O(\lg n)$.

## 3. Основное содержание

### 3.1 Мотивация: зачем нужны деревья поиска

#### Определение / механизм

Рассмотрим задачу: есть база данных с индексом по полю `cost`. При статичном наборе данных можно хранить отсортированный массив и применять **бинарный поиск** за $O(\lg n)$. Однако при **вставке** нового элемента в отсортированный массив/список приходится сдвигать $O(n)$ элементов — это неприемлемо.

Бинарное дерево поиска решает проблему: вставка и поиск выполняются за $O(h)$, где $h$ — высота дерева.

#### Применение BST

- Индексация в базах данных.
- Синтаксический парсинг (деревья разбора).
- Decision Tree (дерево решений) в машинном обучении.

---

### 3.2 Бинарное дерево поиска (BST)

#### Определение

**Бинарное дерево поиска** — дерево, в котором для каждого узла $x$ выполняется:
- все ключи в левом поддереве $< x.\text{key}$,
- все ключи в правом поддереве $> x.\text{key}$.

#### Структура узла

```c
struct Node {
    int key;
    Node* parent;
    Node* left_child;   // NULL, если нет левого потомка
    Node* right_child;  // NULL, если нет правого потомка
};
```

#### Интуиция

Свойство BST позволяет на каждом шаге поиска отбрасывать половину (или больше) дерева, двигаясь влево при $k < x.\text{key}$ и вправо иначе. В идеально сбалансированном дереве это даёт $O(\lg n)$ шагов.

#### Вырожденный случай

Если элементы вставляются в отсортированном порядке, дерево превращается в «цепочку» (связный список), и все операции деградируют до $O(n)$:

```
50
  \
   75
     \
      85
        \
         95
           \
            99
```

В таком дереве $N = K$ (размер равен высоте).

#### Типичные ошибки

- Забыть обновить указатель `parent` при вставке или удалении.
- Считать, что BST всегда сбалансирован — это не так без специальных механизмов.
- Путать $h$ (высоту) и $\lg n$: высота зависит от порядка вставок.

---

### 3.3 Обходы бинарного дерева

Для дерева существует три классических обхода. Порядок вывода зависит от того, когда обрабатывается корневой узел относительно поддеревьев.

#### Центрированный (симметричный) обход — Inorder

**Порядок:** левое поддерево → корень → правое поддерево.  
**Результат:** ключи в отсортированном порядке по возрастанию.  
**Сложность:** $\Theta(n)$

```c
void inorder_tree_walk(Node* x) {
    // Центрированный (симметричный)
    // Ключ в корне выводится между значениями ключей левого и правого поддеревьев.
    if (x != nullptr) {
        inorder_tree_walk(x->left);
        std::cout << x->key << " ";
        inorder_tree_walk(x->right);
    }
}
```

#### Прямой обход — Preorder

**Порядок:** корень → левое поддерево → правое поддерево.  
**Результат:** корень всегда первый; используется для копирования дерева.  
**Сложность:** $\Theta(n)$

```c
void preorder_tree_walk(Node* x) {
    // Обход в прямом порядке.
    // Сначала выводится корень, а затем значения левого и правого поддеревьев.
    if (x != nullptr) {
        std::cout << x->key << " ";
        preorder_tree_walk(x->left);
        preorder_tree_walk(x->right);
    }
}
```

#### Обратный обход — Postorder

**Порядок:** левое поддерево → правое поддерево → корень.  
**Результат:** корень всегда последний; используется для удаления дерева.  
**Сложность:** $\Theta(n)$

```c
void postorder_tree_walk(Node* x) {
    // Обход в обратном порядке.
    // Первыми выводятся значения левого и правого поддеревьев, затем корня.
    if (x != nullptr) {
        postorder_tree_walk(x->left);
        postorder_tree_walk(x->right);
        std::cout << x->key << " ";
    }
}
```

#### Пример для дерева `{2, 5, 5, 6, 7, 8}` (корень = 6)

| Обход | Вывод |
|---|---|
| Inorder (симметричный) | `2 5 5 6 7 8` |
| Preorder (прямой) | `6 5 2 5 7 8` |
| Postorder (обратный) | `2 5 5 8 7 6` |

### 3.4 Операции поиска в BST

#### Поиск элемента

**Механизм:** начинаем с корня. Если текущий узел `NIL` или содержит искомый ключ — завершаем. Иначе идём влево (ключ меньше) или вправо (ключ больше).

**Сложность:** $O(h)$, где $h$ — высота дерева.

```
x ← root, k ← ключ поиска
while x != NIL and x.key != k:
    if k < x.key:
        x = x.left
    else:
        x = x.right
return x
```

#### Минимум и максимум

- **Минимум:** идём по левым потомкам до упора. $O(h)$.
- **Максимум:** идём по правым потомкам до упора. $O(h)$.

#### Следующий элемент (Successor)

**Механизм:**
1. Если у узла есть правое поддерево — следующий = минимум правого поддерева.
2. Иначе поднимаемся по дереву вверх, пока не окажемся в левом потомке — родитель этого узла и есть следующий.

```
if x.right != NIL:
    return min(x.right)
else:
    y = x.parent
    while y != NIL and x == y.right:
        x = y
        y = y.parent
    return y
```

**Сложность:** $O(h)$.

#### Предыдущий элемент (Predecessor)

Симметрично successor: если есть левое поддерево — максимум левого поддерева. Иначе поднимаемся до тех пор, пока не окажемся в правом потомке.

### 3.5 Вставка в BST

**Механизм:** находим место вставки, спускаясь по дереву как в поиске. Новый узел всегда становится листом.

**Сложность:** $O(h)$.

```c
void insert(Tree* tree, Node* z) {
    Node* parent = nullptr;
    Node* current = tree->root;
    while (current != nullptr) {
        parent = current;
        current = z->key < current->key ?
                current->left : current->right;
    }
    z->parent = parent;
    if (parent == nullptr)
        tree->root = z;
    else if (z->key < parent->key)
        parent->left = z;
    else
        parent->right = z;
}
```

#### Интуиция

Алгоритм сохраняет указатель на предыдущий узел (`parent`), чтобы иметь возможность «подвесить» новый лист.

#### Типичные ошибки

- Не обновить `z->parent` после цикла.
- Не обработать случай, когда дерево пустое (`parent == nullptr`).

---

### 3.6 Удаление узла из BST

Удаление — наиболее сложная операция. Рассматриваются четыре случая:

| Случай | Описание | Действие |
|---|---|---|
| 1 | У $z$ нет левого потомка | Заменить $z$ правым потомком (или `NIL`) |
| 2 | У $z$ нет правого потомка | Заменить $z$ левым потомком |
| 3 | У $z$ есть оба потомка, и правый потомок — непосредственный правый ребёнок | Поднять правого потомка вместе с его правым поддеревом |
| 4 | У $z$ есть оба потомка, и в правом поддереве есть ещё узлы | Найти минимум правого поддерева $y$; заменить $z$ на $y$; перенести правое поддерево $y$ |

**Вспомогательная функция `transplant`** — заменяет поддерево с корнем $u$ поддеревом с корнем $v$:

```c
void transplant(Tree* tree, Node* u, Node* v) {
    if (u->parent == nullptr)
        tree->root = v;
    else if (u == u->parent->left)
        u->parent->left = v;
    else
        u->parent->right = v;
    if (v != nullptr)
        v->parent = u->parent;
}
```

**Сложность удаления:** $O(h)$.

### 3.7 Теорема 12.4: Случайное построение BST

#### Формулировка

> Математическое ожидание высоты случайно построенного бинарного дерева поиска с $n$ различными ключами равно $O(\lg n)$.

#### Доказательство (скетч)

**Обозначения:**
- $X_n$ — высота дерева с $n$ ключами.
- $Y_n = 2^{X_n}$ — «экспоненциальная высота».
- $R_n$ — ранг корневого ключа (его позиция в отсортированном множестве).

**Ключевые соотношения:**

Если $R_n = i$, то левое поддерево содержит $i-1$ ключей, правое — $n-i$ ключей, и:
$$Y_n = 2 \cdot \max(Y_{i-1},\, Y_{n-i})$$

Так как $R_n$ равномерно распределён на $\{1, 2, \ldots, n\}$, то $\Pr\{R_n = i\} = \frac{1}{n}$.

Вводим индикаторные переменные $Z_{n,i} = \mathbf{1}[R_n = i]$, для которых $E[Z_{n,i}] = \frac{1}{n}$.

По линейности математического ожидания и независимости $Z_{n,i}$ от $Y_{i-1}$ и $Y_{n-i}$:
$$E[Y_n] \leq \frac{2}{n} \sum_{i=1}^{n} E[Y_{i-1}]$$

Решая рекуррентное соотношение, получаем, что $E[Y_n]$ растёт полиномиально по $n$, откуда:
$$E[X_n] = E[\log_2 Y_n] = O(\lg n)$$

**Итог:** В среднем случае BST работает за $O(\lg n)$, но в худшем — за $O(n)$. Именно для гарантии $O(\lg n)$ в худшем случае нужны красно-чёрные деревья.

### 3.8 Красно-чёрные деревья (RBT)

#### Определение

**Красно-чёрное дерево** — приближённо сбалансированное бинарное дерево поиска, в котором каждый узел покрашен в красный или чёрный цвет, и выполняются пять свойств.

#### Структура узла

```c
struct RBNode {
    int key;
    char color;      // 'R' или 'B'
    RBNode* parent;
    RBNode* left;
    RBNode* right;
};
```

#### Пять свойств красно-чёрного дерева

1. Каждый узел является либо красным, либо чёрным.
2. Корень дерева является чёрным узлом.
3. Каждый лист дерева (`NIL`) является чёрным узлом.
4. Если узел красный, то оба его дочерних узла чёрные (нет двух красных узлов подряд).
5. Для каждого узла все простые пути от него до листьев-потомков содержат одинаковое количество чёрных узлов.

#### Понятие чёрной высоты

**Черная высота** $bh(x)$ — количество чёрных узлов на любом простом пути от узла $x$ (не считая сам $x$) до листа.

#### Интуиция

Свойство 4 не позволяет создать «цепочку» из красных узлов. Свойство 5 гарантирует, что чёрные узлы распределены равномерно. Вместе они не позволяют одному пути быть более чем вдвое длиннее другого.

---

### 3.9 Лемма 13.1: Ограничение высоты RBT

#### Формулировка

> Красно-чёрное дерево с $n$ внутренними узлами имеет высоту, не превышающую $2\lg(n+1)$.

#### Доказательство

**Шаг 1.** Покажем, что поддерево любого узла $x$ содержит не менее $2^{bh(x)} - 1$ внутренних узлов (индукцией по высоте $x$).

- **База:** высота $x = 0$ (лист). Число узлов $= 2^0 - 1 = 0$. Верно.
- **Переход:** каждый дочерний узел имеет чёрную высоту $bh(x)$ (если он красный) или $bh(x) - 1$ (если чёрный). По индукции каждый потомок содержит $\geq 2^{bh(x)-1} - 1$ узлов. С корнем $x$:
$$\text{узлы}(x) \geq (2^{bh(x)-1} - 1) + (2^{bh(x)-1} - 1) + 1 = 2^{bh(x)} - 1$$

**Шаг 2.** Пусть $h$ — высота дерева. По свойству 4, как минимум половина узлов на пути от корня до листа — чёрные. Значит:
$$bh(\text{root}) \geq h/2$$

**Шаг 3.** Из шага 1:
$$n \geq 2^{h/2} - 1 \implies n + 1 \geq 2^{h/2} \implies h \leq 2\lg(n+1)$$

#### Следствие

Операции поиск, min, max, successor, predecessor выполняются за $O(\lg n)$ в **наихудшем** случае.

### 3.10 Вставка в красно-чёрное дерево

#### Общая схема

1. Вставляем узел $z$ как в обычное BST, окрашиваем его в **красный**.
2. Исправляем нарушения свойств (прежде всего свойства 4 — два красных подряд) с помощью перекраски и поворотов.

#### Три случая (по Кормену)

Нарушение происходит, когда вставленный узел $z$ красный, и его родитель $P$ тоже красный. Пусть $G$ — дед, $U$ — дядя (брат $P$).

**Случай 1: Дядя $U$ красный**
- $P$ и $U$ перекрашиваем в чёрный.
- $G$ перекрашиваем в красный.
- Повторяем проверку для $G$ как нового $z$ (поднимаемся вверх).

**Случай 2: Дядя $U$ чёрный, $z$ — «треугольник»** (т.е. $z$, $P$ и $G$ не лежат на одной прямой)
- Поворот вокруг $P$ (чтобы $z$ стал внешним узлом).
- Переходим к случаю 3.

**Случай 3: Дядя $U$ чёрный, $z$ — «линия»** (т.е. $z$, $P$ и $G$ на одной прямой)
- Поворот вокруг $G$ ($P$ поднимается вверх).
- $P$ перекрашиваем в **чёрный**, $G$ — в **красный**.

#### Сложность вставки

$O(\lg n)$ — высота дерева; количество поворотов — не более двух.

#### Пример вставки (из презентации)

Вставка узла со значением 9 в существующее дерево:

```
До вставки:                 После вставки и исправления:
    ...                           ...
    8 (чёрный)                    9 (чёрный)
   / \                           / \
  6   ...                       8   ...
                                /
                               6
```

[дополнено]
При вставке подряд значений 9, 10, 12, 13 дерево может потребовать нескольких поворотов и перекрасок. Ключевой момент: после каждой вставки нужно подниматься от вставленного узла к корню и устранять нарушения. В худшем случае количество перекрасок — $O(\lg n)$, поворотов — не более 2.
[/дополнено]

### 3.11 Удаление из красно-чёрного дерева

Удаление сложнее вставки: при удалении чёрного узла нарушается свойство 5 (равенство чёрных высот). Для исправления используется понятие «двойной чёрный» узел и четыре подслучая. Лектор отметил это как домашнее задание (источник: слайд «Удаление из красно-черных деревьев — это как домашнее задание»).

[дополнено]
**Основная идея удаления:**
1. Удаляем узел как в обычном BST (с использованием `transplant`).
2. Если удалялся чёрный узел — появляется «двойной чёрный» дефицит.
3. Исправляем дефицит, перебирая 4 случая в зависимости от цвета брата и его детей. В худшем случае — $O(\lg n)$ поворотов.

**Сложность удаления:** $O(\lg n)$.
[/дополнено]

### 3.12 Система непересекающихся множеств (DSU / Union-Find)

> **Примечание:** этот раздел основан на транскрипции лекции. Тема DSU тесно связана с деревьями и рассматривается как смежная структура данных.

#### Определение

**Система непересекающихся множеств (Disjoint Set Union, DSU)** — структура данных, поддерживающая разбиение универсального множества на попарно непересекающиеся подмножества.

#### Три основные операции

| Операция | Описание | Сложность (базовая) |
|---|---|---|
| `make_set(x)` | Создать новое множество из одного элемента $x$ | $O(1)$ |
| `find(x)` | Вернуть представителя (корень) множества, содержащего $x$ | Зависит от реализации |
| `union(x, y)` | Объединить множества, содержащие $x$ и $y$ | Зависит от реализации |

#### Реализация 1: Хэш-таблица + связные списки

Словарь (хэш-таблица) хранит ссылки на «голову» (опорный элемент) каждого множества. Каждое множество — связный список. Дополнительно хранится указатель на хвост для быстрого добавления.

**Сложность `union` (наивная):** добавляем все элементы меньшего множества к большему: $O(n)$ в худшем случае.

**Эвристика «объединение по размеру»:** всегда добавляем меньшее множество к большему. Это не меняет $O(n)$ в худшем случае для одной операции, но улучшает амортизированную сложность.

#### Реализация 2: Лес деревьев (оптимальная)

Каждое множество хранится как дерево. Корень дерева — представитель множества. Указатель `parent` направлен к корню.

```c
struct Node {
    int val;
    Node* parent;
    int size;   // опционально
    int rank;   // опционально
};
```

**Правило:** корень дерева указывает сам на себя (`parent == self`).

#### Эвристика 1: Объединение по рангу (Union by Rank)

Ранг — величина, похожая на высоту дерева (но не всегда совпадающая с ней). При объединении дерево с меньшим рангом подвешивается под корень дерева с большим рангом.

**Гарантия:** высота дерева не превышает $O(\lg n)$.

#### Эвристика 2: Сжатие пути (Path Compression)

При выполнении `find(x)` все узлы на пути от $x$ до корня напрямую перевешиваются на корень.

```python
def find(x):
    if x.parent != x:
        x.parent = find(x.parent)  # рекурсивно сжимаем путь
    return x.parent
```

#### Амортизационная сложность с двумя эвристиками

При совместном применении объединения по рангу и сжатия пути:

$$\text{амортизированная сложность одной операции} = O(\alpha(n))$$

где $\alpha(n)$ — **обратная функция Аккермана**, которая растёт настолько медленно, что для любых практических значений $n \leq 10^{80}$ выполняется $\alpha(n) \leq 5$. На практике это фактически $O(1)$.

#### Применение DSU

- **Алгоритм Краскала** (минимальное остовное дерево): объединяем вершины, соединённые добавляемым ребром.
- **Задачи на связность графа**: проверка, принадлежат ли две вершины одной компоненте связности.
- **Кластеризация точек** на плоскости.

#### Типичные ошибки

- Не инициализировать `parent[x] = x` при `make_set`.
- Забыть обновить ранг при объединении равных по рангу деревьев.
- Использовать наивную реализацию без эвристик в задачах с большим числом запросов.

## 5. Справочный лист

### Сложности BST

- Поиск: $O(h)$
- Вставка: $O(h)$
- Удаление: $O(h)$
- Min / Max: $O(h)$
- Successor / Predecessor: $O(h)$
- Inorder / Preorder / Postorder обход: $\Theta(n)$
- $h = O(\lg n)$ в среднем (Теорема 12.4); $h = O(n)$ в худшем случае

### Сложности красно-чёрного дерева

- Высота: $h \leq 2\lg(n+1)$
- Все словарные операции: $O(\lg n)$ в **наихудшем** случае
- Вставка: $O(\lg n)$, максимум 2 поворота
- Удаление: $O(\lg n)$, максимум 3 поворота

### Пять свойств красно-чёрного дерева (список для запоминания)

1. Цвет: красный или чёрный
2. Корень — чёрный
3. Листья (`NIL`) — чёрные
4. Красный узел имеет только чёрных детей
5. Все пути к листьям содержат одинаковое число чёрных узлов

### Сложности DSU

| | Без эвристик | Union by Rank | Path Compression | Оба |
|---|---|---|---|---|
| `find` | $O(n)$ | $O(\lg n)$ | $O(\lg n)$ амортиз. | $O(\alpha(n))$ амортиз. |
| `union` | $O(n)$ | $O(\lg n)$ | $O(\lg n)$ амортиз. | $O(\alpha(n))$ амортиз. |

### Ключевые факты

- Inorder-обход BST выдаёт ключи в отсортированном порядке.
- Минимум BST — самый левый узел; максимум — самый правый.
- В RBT `NIL`-листья считаются чёрными и часто реализуются как один общий «страж» (`sentinel`).
- $\alpha(n) \leq 5$ для всех практически встречающихся $n$ (вплоть до числа атомов во Вселенной).
- Алгоритм Краскала строит MST за $O(E \lg E)$ с использованием DSU.

## 6. Практические советы

1. **Никогда не используйте обычный BST без балансировки в production-коде**, если данные могут поступать в отсортированном или почти отсортированном порядке — дерево деградирует до связного списка. Используйте RBT, AVL-дерево или `std::map` (который в большинстве реализаций является RBT).

2. **При реализации DSU всегда применяйте обе эвристики** (объединение по рангу + сжатие пути) вместе. Каждая по отдельности даёт $O(\lg n)$ амортизационно, а вместе — $O(\alpha(n))$.

3. **Для задач на MST (минимальное остовное дерево) предпочитайте алгоритм Краскала с DSU**, а не Прима с приоритетной очередью, если граф задан списком рёбер, а не матрицей смежности.

4. **Помните про случай вырожденного дерева**: при написании рекурсивных обходов или итеративных алгоритмов всегда проверяйте граничные случаи — пустое дерево (`root == nullptr`) и дерево из одного узла.

5. **При реализации операций с RBT держите под рукой все три случая вставки (по Кормену)**: случай 1 (дядя красный) решается перекраской и подъёмом; случаи 2 и 3 (дядя чёрный) решаются поворотами. Случаи 2 и 3 всегда завершают цикл исправлений (не требуют подъёма выше).
