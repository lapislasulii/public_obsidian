

### 1. Переход к мощным абстракциям

Ранее программирование строилось на линейном подходе: команды выполнялись шаг за шагом с использованием переменных, условий и циклов. Однако с ростом сложности программ требуется **абстракция** — скрытие сложных деталей реализации за простым интерфейсом.

- **Абстракция кода:** упаковка повторяющихся блоков в **функции**.
- **Абстракция данных:** объединение связанных данных в **структуры данных** или коллекции.

---

### 2. Функции: строительные блоки кода

**Функция** — это именованный блок кода, предназначенный для выполнения конкретной задачи. Сравнить это можно с книгой рецептов: вместо того чтобы каждый раз описывать, как взбивать яйца, вы пишете инструкцию один раз и просто ссылаетесь на неё.

**Преимущества функций:**

- **Повторное использование:** пишем один раз, вызываем многократно, избегая копирования кода.
- **Модульность:** программа делится на независимые, понятные части.
- **Читаемость:** понятные названия (например, `calculate_tax()`) объясняют суть действий.

**Основы работы с функциями:**

- **Определение:** начинается с ключевого слова `def`. Само по себе определение не запускает код; он выполняется только при **вызове** функции по имени со скобками.
- **Параметры и аргументы:** параметры — это «коробки» (переменные) в определении функции, а аргументы — это реальные значения, которые мы в них кладём при вызове.
- **Типы аргументов:**
    1. **Позиционные:** передаются в том же порядке, что и параметры.
    2. **Именованные (Keyword):** явно указывается имя параметра, порядок не важен.
    3. **Значения по умолчанию:** задаются при определении. Если аргумент не передан, используется дефолтное значение.

**Оператор `return`:** Функции могут возвращать результат. Оператор `return` немедленно завершает выполнение и отправляет значение обратно в место вызова.

- **Print vs Return:** `print` — это побочный эффект (просто вывод на экран), а `return` — это фактический результат, который можно сохранить в переменную или использовать в расчётах.
- Функции без `return` неявно возвращают значение `None`.
- Python позволяет возвращать **несколько значений** одновременно, упаковывая их в кортеж.

**Область видимости (Scope):**

- **Локальные переменные:** создаются внутри функций и невидимы снаружи.
- **Глобальные переменные:** создаются вне функций и доступны для чтения отовсюду.
- _Рекомендация:_ избегайте использования ключевого слова `global` для изменения переменных; лучше передавайте данные через параметры и возвращайте через `return`.

### 3. Структуры данных (Коллекции)

Когда простых переменных недостаточно (например, для списка покупок или телефонной книги), используются коллекции.

- **Списки (Lists):** упорядоченные и изменяемые.
    - Индексация начинается с 0; поддерживаются отрицательные индексы (–1 — последний элемент).
    - Методы: `append()` (добавить в конец), `insert()` (вставить по индексу), `pop()` (удалить по индексу), `remove()` (удалить по значению), `sort()` и `reverse()`.
- **Кортежи (Tuples):** упорядоченные, но **неизменяемые**.
    - Используются для защиты данных от случайных изменений, для хранения констант и в качестве ключей словарей.
    - Занимают меньше памяти и работают быстрее списков.
- **Словари (Dictionaries):** коллекции пар «ключ-значение».
    - Доступ к данным осуществляется по уникальному ключу, а не по индексу.
    - Ключи должны быть уникальными и неизменяемыми.
- **Множества (Sets):** неупорядоченные коллекции **уникальных** элементов.
    - Идеальны для удаления дубликатов и быстрой проверки наличия элемента.
    - Поддерживают математические операции: объединение (`|`), пересечение (`&`) и разность (`-`).

### 4. Работа с файлами и модулями

Для того чтобы данные не исчезали после завершения программы, используется файловая система.

- **Файлы:** функция `open()` используется для работы с файлами. Рекомендуется использовать конструкцию `with open(...)`, которая гарантирует закрытие файла.
    - **Режимы:** `'r'` (чтение), `'w'` (запись с перезаписью), `'a'` (добавление в конец).
- **Модули:** позволяют использовать готовый код из стандартной библиотеки Python (например, `math`).
    - Можно импортировать весь модуль (`import math`) или конкретные функции (`from math import sqrt`).
- **JSON:** стандартный формат для сохранения сложных данных (списков и словарей) в текстовом виде с помощью модуля `json` (функции `json.dump` и `json.load`).

### 5. Итоговый проект и развитие

В качестве практики предлагается создать **Улучшенную телефонную книгу**, которая объединяет все изученные темы: функции для логики, словари для данных и JSON для сохранения состояния между запусками.

**Дальнейшие шаги в обучении:**

1. **Обработка ошибок:** использование `try...except` для предотвращения сбоев программы.
2. **ООП (Объектно-ориентированное программирование):** объединение данных и функций в единые сущности — объекты.
3. **Сторонние библиотеки:** использование `pip` для установки инструментов для веб-разработки, анализа данных и создания игр.
